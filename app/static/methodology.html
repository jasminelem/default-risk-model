Methodology

A detailed overview of the data sources, machine learning pipeline, and explainability framework used to build this model.

1. Data Pipeline & Sources

The foundation of this model is a clean, time-series dataset constructed from multiple financial databases. The process ensures no data leakage and respects the temporal nature of financial reporting.

DATA SOURCES (WRDS)
  • Compustat Fundamentals (funda/fundq): Annual and quarterly financial statements (balance sheet, income statement).
  • CRSP Stock Prices (dsf): Daily stock returns and prices used to calculate market volatility and momentum.
  • FJC Bankruptcy Database: Historical records of corporate bankruptcy filings used to define the target variable.
  • Macroeconomic Data (FRED): Interest rates (Fed Funds, Treasury yields) and credit spreads (Baa/Aaa) as external risk factors.
PIPELINE FLOW
Raw Data → Linking (GVKEY/PERMNO) → Feature Engineering (60+ ratios) → Time-Split (Train/Val/Test) → Final Datasets

2. Model Training & Architecture

We trained two separate XGBoost Classifier models to predict the probability of default over two different horizons.

12-Month Model
Target: default_in_next_12m
Optimised via Optuna (500+ trials) to maximise ROC-AUC on a time-based validation set.
5-Year Model
Target: default_in_next_5y
Optimised via Optuna (500+ trials) to maximise ROC-AUC on a time-based validation set.
The models use 60+ engineered features covering leverage, profitability, liquidity, market signals, and macroeconomic conditions. Hyperparameters are tuned to prevent overfitting on historical data.

3. Explainability (SHAP)

To move beyond a "black box," we use SHAP (SHapley Additive exPlanations) to interpret model outputs.

Global View: We generate summary plots (beeswarm/bar) to show which features are most important for the model overall and in which direction they push the prediction.

Local View: For every individual company prediction, we calculate the exact contribution (SHAP value) of every feature. A positive value pushes the prediction towards "Default," a negative value pushes it towards "No Default."

The interactive chart on the main dashboard shows the local SHAP values for a specific company. The slider lets you compare the drivers for the 12-month vs 5-year horizons.

4. Deployment

The application is deployed as a containerised service:
  • Backend: A FastAPI server loads the pre-trained XGBoost models and the company index into memory for low-latency inference.
  • Frontend: A static HTML/JS interface served by the same container, consuming the backend API for predictions and SHAP explanations.
  • Data: Large data files (Parquets) are managed via Git LFS to ensure reproducibility.
